# API Context

> The HTTP and SSE surface that the frontend (and Stage B partners) consume.
> REST + OpenAPI only; no GraphQL, no gRPC at the public boundary.

---

## TL;DR — 10 lines

1. **REST + OpenAPI only.** GraphQL and gRPC are forbidden at the public
   surface (blueprint §17). gRPC is allowed internally between AI runtime
   and the L2 tool-runner sidecar.
2. **OpenAPI spec is generated by FastAPI** at `/openapi.json`. The frontend
   regenerates types via `frontend/scripts/check-sdk-drift.ts`. Drift
   between server and generated types fails CI.
3. **Three generation entry paths** under `/api/generate/`: `jobs` (queued,
   Temporal-backed; canonical), `stream` (token streaming when enabled),
   `sync` (legacy synchronous fallback).
4. **SSE is the only push channel.** Endpoint:
   `GET /api/generate/agentic-stream/{job_id}`. Header `X-Session-ID` is
   set on the response so clients can resume via `Last-Event-ID` semantics.
   The replay endpoint is
   `GET /pipeline/agentic-stream/{session_id}/replay?after_sequence=N`
   (P0-7 SHIPPED — m12-pr05).
5. **Idempotency-Key middleware** is mandatory on POST/PATCH/DELETE
   (P0-6 SHIPPED). Clients send a UUID; replays return the cached response
   for 24h.
6. **Rate limiting** via slowapi (Redis-backed). 246 `@limiter.limit(...)`
   decorators across 46/48 route files. Health and metrics are intentionally
   unlimited.
7. **Auth** is Supabase JWT in `Authorization: Bearer <token>`. RLS in
   Postgres scopes every query to the user's `org_id`.
8. **Errors** are returned as `application/json` with `{detail, code,
   correlation_id}`. Never leak Postgres / LLM stack traces. Sentry
   `before_send` scrubs PII at depth 16 (TD-2 SHIPPED).
9. **Event taxonomy** lives in
   [`packages/events/schema/v1/`](../packages/events/schema/v1) — ~30
   canonical types with JSON Schema. Codegen produces Python / TS / Kotlin
   types (P1-7 SHIPPED).
10. **Versioning:** `/api/v1/...` reserved (Stage B). Today everything is
    unversioned but treated as v1; breaking changes require an ADR + a
    migration window with both versions live.

## URL conventions

| Pattern | Use |
|---|---|
| `/api/<domain>/...` | All public REST routes (Stage A). Stage B will mount under `/api/v1/`. |
| `/api/generate/...` | Generation pipeline entry + control. |
| `/api/generate/agentic-stream/{id}` | SSE for live Mission-Control. |
| `/pipeline/agentic-stream/{id}/replay` | SSE replay (note: NOT under `/api`). |
| `/webhooks/<source>` | Inbound webhooks (Stripe, Supabase). No JWT; signed by source. |
| `/health`, `/ready` | Liveness / readiness probes. No auth, no rate limit. |
| `/metrics` | Prometheus text format. No auth, no rate limit. Bound to private network in Stage B. |
| `/openapi.json`, `/docs`, `/redoc` | OpenAPI + Swagger UI + Redoc. |

## Standard request shape

```http
POST /api/applications
Authorization: Bearer <supabase-jwt>
Idempotency-Key: 7d6e0c5a-...
Content-Type: application/json

{
  "role_title": "Senior SRE",
  "company": "Acme",
  "jd_text": "..."
}
```

```http
HTTP/1.1 201 Created
Content-Type: application/json
X-Correlation-ID: 1e2f...

{ "id": "app_...", "status": "draft", "created_at": "2026-05-08T..." }
```

Conventions:

- Body and response are Pydantic models (`<Verb><Resource>Request`,
  `<Resource>Response`).
- Errors: `4xx { "detail": "...", "code": "billing.cap_exceeded",
  "correlation_id": "..." }`.
- All timestamps ISO 8601 in UTC.
- All IDs are prefixed (`app_`, `job_`, `usr_`, `org_`, `ev_`).
- `org_id` is never sent in the body — it is read from the JWT.

## Idempotency contract (P0-6 SHIPPED)

Every state-changing request (POST/PATCH/DELETE) MUST include an
`Idempotency-Key` header. The middleware:

1. Hashes the route + key + body to a cache row in `idempotency_keys`.
2. If a cache row exists and is `<24h` old, returns the cached response.
3. Otherwise, runs the handler and caches the response on success
   (status `2xx`).

Clients are expected to:

- Generate a UUIDv4 per logical action (not per retry).
- Reuse the same key when retrying after a network failure.
- Treat `Idempotency-Key conflict (409)` as a body mismatch — same key,
  different body — and surface as a developer error.

## SSE contract (P0-7 SHIPPED — m12-pr05)

### Initial subscribe

```http
GET /api/generate/agentic-stream/{job_id}
Authorization: Bearer <supabase-jwt>
Accept: text/event-stream

HTTP/1.1 200 OK
Content-Type: text/event-stream
X-Session-ID: sess_8f2a...
Cache-Control: no-cache, no-transform
```

The server emits SSE frames:

```
id: 1
event: pipeline.started
data: {"job_id":"job_...","phase":"Recon"}

id: 2
event: stage.started
data: {"phase":"Atlas","stage":"resume_parse"}

id: 3
event: stage.token
data: {"phase":"Atlas","text":"Extracted skill: Kubernetes\n"}

...

id: N
event: pipeline.completed
data: {"job_id":"job_...","status":"ok","artifacts":["cv","cover","plan"]}
```

### Resumption

When the connection drops, the client reconnects with the same
`X-Session-ID` and a `Last-Event-ID` header (the highest `id:` it saw):

```http
GET /pipeline/agentic-stream/{session_id}/replay?after_sequence=42
Authorization: Bearer <supabase-jwt>
Accept: text/event-stream
```

`AgenticEventEmitter.get_events_after(after_sequence)` returns the missed
events in order; the client resumes consumption transparently.

**Limitation today:** the in-memory store is single-process. Cross-pod
replay requires the Redis/DB-backed session registry — tracked in
blueprint §18 P0-7 follow-up.

**Frontend contract:** native `EventSource` is forbidden because it does
not support custom headers (and therefore cannot send the JWT). Use
[`frontend/src/lib/sseClient.ts`](../frontend/src/lib/sseClient.ts) which
uses `fetch` with a streaming body (ADR-0039 Proposed).

## Event taxonomy (~30 canonical types)

JSON schemas live in [`packages/events/schema/v1/`](../packages/events/schema/v1/).
Codegen (`packages/events/scripts/codegen.py`) produces:

- Python: `packages/events/python/`
- TypeScript: `packages/events/typescript/`
- Kotlin: `packages/events/kotlin/` (mobile target)

P1-2 SHIPPED (m7-pr31): `OutboxWriter.append` validates the payload
against the schema for the named event type. Unknown type or schema
mismatch raises immediately.

Five v1 events shipped today (the rest exist as schemas and emit when
their producer surface ships):

- `aim.assignment.created`
- `aim.source.created`
- `generation.requested`
- `generation.completed`
- `mission.draft.created`

Pipeline streaming events (frontend-only — not in outbox):

- `pipeline.started`, `pipeline.completed`, `pipeline.failed`
- `stage.started`, `stage.token`, `stage.completed`, `stage.error`
- `agent.message`, `agent.tool_call`, `agent.tool_result`

Contract test: `backend/tests/contracts/test_event_schema_contract.py`
asserts every event type used in code has a schema.

## Error catalogue (selected)

| Code | HTTP | Meaning |
|---|---|---|
| `auth.invalid_token` | 401 | JWT signature / expiry / audience |
| `auth.scope_missing` | 403 | role lacks the required scope |
| `billing.cap_exceeded` | 402 | per-org daily $ cap (P0-4) |
| `billing.fail_closed` | 503 | upstream Supabase outage; production fail-closed (TD-7) |
| `idempotency.conflict` | 409 | same key, different body |
| `rate_limit.exceeded` | 429 | slowapi token bucket exhausted; `Retry-After` header set |
| `pipeline.unavailable` | 503 | all three execution paths failed |
| `validation.error` | 422 | Pydantic validation failure |
| `not_found` | 404 | row missing (RLS-aware: same as forbidden to avoid existence leaks) |

Routes never raise raw `Exception`; map to `HTTPException` with these
codes.

## OpenAPI tags

Frontend OpenAPI SDK is grouped by tag for ergonomic imports
(`api.applications.list()`). Tags are declared per route module; the
canonical tag list is in `backend/main.py` `app.openapi_tags`.

## API + frontend SDK drift

The script `frontend/scripts/check-sdk-drift.ts`:

1. Boots backend in test mode.
2. Fetches `/openapi.json`.
3. Runs `openapi-typescript` to regenerate `frontend/src/lib/api/generated.d.ts`.
4. Diffs against the committed file.
5. Exits non-zero on diff.

CI gate `openapi-drift` runs this. PRs that change a route signature must
also commit the regenerated SDK.

## Webhook handling

`webhooks.py` accepts inbound webhooks from third parties (Stripe,
Supabase, future GitHub). Each route:

- Verifies the source's signature header.
- Bypasses JWT auth.
- Bypasses idempotency middleware (sources have their own retry semantics).
- Writes to `events_outbox` for downstream processing; never does work
  inline beyond signature verification + outbox append.

## What "good API" looks like in this repo

- [ ] Pydantic request and response models, both named.
- [ ] `Depends(get_current_user)` and `Depends(get_db)` on every
      authenticated route.
- [ ] `@limiter.limit("60/minute")` on every route (or stricter where
      sensible).
- [ ] Idempotency-Key respected on POST/PATCH/DELETE.
- [ ] Errors raise typed exceptions mapped to the catalogue above.
- [ ] No raw SQL; use the service layer (which uses supabase client).
- [ ] OpenAPI tag declared.
- [ ] Frontend SDK regenerated and committed if route signature changed.
- [ ] Contract test updated if a new event type emitted.
